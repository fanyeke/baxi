## 1. 概述

v0.5 将 Olist 巴西电商数据决策沙盘从 **"直接访问 SQLite + 手动跑脚本"** 升级为 **"FastAPI 本地 API 服务"**，使 Qoder（LLM Agent）可以通过受控的 HTTP API 调用系统，而不需要：

- 进入项目目录
- 直接读写 SQLite
- 手动执行 Python 脚本

### 架构变化

```
Before (v0.2-v0.4):
  Qoder / 人 → 进入项目 → 执行脚本 → 读写 SQLite → 同步飞书

After (v0.5):
  Qoder / 人 → 调用 Local API → Bearer Token 认证 → API 服务层
    → services/ 包 → SQLite / Adapter → 飞书同步
    → 结构化 JSON 响应 + 错误诊断
```

### 2.2 api/ FastAPI 层

| 模块 | 职责 |
|------|------|
| `api/main.py` | FastAPI app 工厂，request_id 中间件，异常处理器注册，路由挂载，启动迁移 |
| `api/errors.py` | `APIError` 异常类，9 个错误码，结构化 JSON 错误响应 |
| `api/auth.py` | Bearer Token 验证 |
| `api/dependencies.py` | `get_db()` 数据库连接依赖，`get_current_user()` 认证依赖 |
| `api/schemas.py` | Pydantic v2 模型：HealthResponse, StatusResponse, AlertItem/List, TaskItem/List, OutboxItem/List, DispatchRequest/Response, ErrorResponse |
| `api/logging_config.py` | 3 个 JSON 日志器（api.log, error.log, audit.log） |

### 2.3 6 个 P0 API 端点

| 端点 | 方法 | 认证 | 功能 |
|------|------|------|------|
| `/api/v1/health` | GET | 无 | 健康检查：状态 + 数据库连接 |
| `/api/v1/status` | GET | Bearer | 系统状态：12 表行数 + 最近管线运行 |
| `/api/v1/alerts` | GET | Bearer | 告警查询：按 severity/status/object_type 过滤 |
| `/api/v1/tasks` | GET | Bearer | 任务查询：按 status/priority/owner_role 过滤 |
| `/api/v1/outbox` | GET | Bearer | Outbox 查询：按 status/channel 过滤 |
| `/api/v1/outbox/dispatch` | POST | Bearer | Outbox 分发：默认 dry-run，显式 apply |

### 2.4 错误契约

所有错误响应统一格式：

```json
{
  "request_id": "uuid",
  "error_code": "AUTH_REQUIRED",
  "message": "Authorization header is required",
  "diagnosis": "No Bearer token provided",
  "suggested_action": "Add 'Authorization: Bearer <token>' header"
}
```

9 个错误码：`AUTH_REQUIRED`, `INVALID_TOKEN`, `DB_UNAVAILABLE`, `DB_QUERY_FAILED`, `OUTBOX_NOT_FOUND`, `DISPATCH_FAILED`, `FEISHU_DISPATCH_FAILED`, `VALIDATION_ERROR`, `CONFIG_MISSING`

### 2.5 日志系统

| 日志文件 | 级别 | 内容 |
|----------|------|------|
| `logs/api/api.log` | INFO | 所有请求（method, path, status_code, duration_ms） |
| `logs/api/error.log` | ERROR | 错误详情（error_code, diagnosis, suggested_action） |
| `logs/api/audit.log` | INFO | 写操作审计 |

外加 `data/system/api_audit_log.csv` 记录每次 dispatch 操作。

## 7. 启动说明

```bash
# 1. 设置 Token
echo 'API_BEARER_TOKEN=your-secret-token' >> .env

# 2. 启动服务
python3 scripts/run_api.py

# 3. 健康检查（无需认证）
curl http://127.0.0.1:8765/api/v1/health

# 4. 带认证查询
curl -H "Authorization: Bearer your-secret-token" \
  "http://127.0.0.1:8765/api/v1/alerts?severity=high&limit=5"

# 5. 分发 Outbox（干跑）
curl -X POST -H "Authorization: Bearer your-secret-token" \
  -H "Content-Type: application/json" \
  -d '{"channel":"manual","limit":1}' \
  http://127.0.0.1:8765/api/v1/outbox/dispatch

# 6. API 文档
open http://127.0.0.1:8765/docs
```
