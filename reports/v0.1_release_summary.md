## 1. 版本定义

v0.1 是 Olist 项目的**首个可运行收敛版本**。核心定位：

- **决策模式**: 规则驱动 / 启发式（decision_source = "heuristic"），不依赖真实大模型
- **数据范围**: Olist 巴西电商 2016-2018 全量数据（634 天，99,441 订单）
- **输出形式**: 本地 CSV/JSON/PNG + 飞书多维表格沙盘（dry-run 验证）
- **代码规模**: 37 个 Python 脚本，压缩为 4 个入口命令

## 6. 已知限制

1. **决策来源为启发式规则**: 所有策略 `decision_source = "heuristic"`，LLM API 在无 key 时回退到规则模式
2. **FROZEN 历史脚本**: Phase 1-7 的 14 个 EDA 脚本路径失效，仅 3 个基础表生成脚本已修复
3. **飞书凭证为占位符**: `.env` 中均为 `YOUR_*`，dry-run 模式可正常运行但不会真实写入
4. **review_retro 为采样数据**: 5-10 条复盘记录基于 hindsight 规则生成，非真实运营反馈
5. **data_quality_checks 非关键 WARN**: 小规模日数据上的质量检查有已知警告，不影响结果
6. **维度级异常检测未实现**: category_decline / region_anomaly / marketing_seller_gap 三类规则 deferred 到 Phase II
