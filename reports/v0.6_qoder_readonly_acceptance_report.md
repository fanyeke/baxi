## 1. Executive Summary

v0.6 introduces a read-only API contract for Qoder (AI Agent), enabling safe, auditable system introspection without triggering any business actions. The feature provides 3 endpoints for Qoder to understand the system state, list its capabilities, and record inspection reports.

**Core Principles Achieved**:
- Qoder can **read** system state but never **execute** business actions
- All write endpoints enforce `no_apply_performed=true`
- No modifications to alerts, tasks, outbox, or config tables
- Bearer Token authentication on all endpoints

## 6. API Endpoints Summary

| Method | Path | Auth | Rate Limit | Description |
|--------|------|------|-----------|-------------|
| GET | `/api/v1/qoder/capabilities` | Bearer Token | 60/60s | Return Qoder capability matrix |
| GET | `/api/v1/qoder/context` | Bearer Token | 60/60s | Aggregated system context |
| POST | `/api/v1/qoder/reports` | Bearer Token | 60/60s | Record inspection report |

### Query Parameters (context)
- `severity` - Filter alerts by severity level
- `limit_alerts` - Max alerts to return (default: 10)
- `limit_tasks` - Max tasks to return (default: 10)
- `limit_outbox` - Max outbox entries to return (default: 10)
- `include_logs` - Whether to include recent log entries (default: false)

### Report Constraints
- `no_apply_performed` MUST be `true` (422 otherwise)
- `run_type` and `summary` are required fields
- Duplicate `report_id` returns success without duplicate insert (idempotent)
- Reports always record `business_side_effect=false`

## 8. Known Limitations

1. **No GET /qoder/reports list endpoint** - Historical report retrieval not available in v0.6. Qoder can POST reports but cannot query past reports via API. Planned for v0.6.1.

2. **No dry-run actions** - Qoder can read context and record reports but cannot simulate actions (e.g., "what if I applied rule X?"). Planned for v0.6.1.

3. **No notification adapters** - Reports and alerts are not pushed to external channels (DingTalk, etc.). Planned for v0.6.2.

4. **No frontend /console/qoder page** - Qoder status and reports are API-only. Frontend deferred.

5. **qoder_service.py 91% coverage** - 6 missed lines in edge case paths (error handling, rare DB states). Acceptable for initial release.

6. **No daily incremental loop** - Qoder does not autonomously run periodic inspections. Planned for v0.7.
