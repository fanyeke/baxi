# v0.2 DB Runbook

> **ARCHIVED:** This runbook documents the legacy v0.2 Python/SQLite infrastructure that has been fully migrated to Go/PostgreSQL. The `python3` commands and scripts referenced below no longer exist. Current operations use Go CLI commands documented in `README.md`.

## Quick Start

```bash
# Initialize database
python3 scripts/db_init.py

# Ingest all data
python3 scripts/db_ingest.py --mode full

# Calculate metrics
python3 scripts/db_calculate_metrics.py --mode full

# Run rule engine
python3 scripts/db_rule_engine.py --mode full

# Export to Feishu CSVs
python3 scripts/db_export_feishu.py --all

# Simulate trigger dispatch
python3 scripts/db_trigger_simulator.py --dry-run

# Run full pipeline (all steps above)
python3 scripts/run_db_pipeline.py --mode full
```

## Commands Reference

| Command | Description |
|---------|-------------|
| `python3 scripts/db_init.py` | Create/recreate database with 12 tables |
| `python3 scripts/db_ingest.py --mode full` | Full ingestion from CSV |
| `python3 scripts/db_ingest.py --mode range --start 2017-11-01 --end 2017-11-30` | Range ingestion |
| `python3 scripts/db_ingest.py --mode date --date 2017-11-24` | Single date ingestion |
| `python3 scripts/db_calculate_metrics.py --mode full` | Compute all daily metrics |
| `python3 scripts/db_rule_engine.py --mode full` | Run all rules |
| `python3 scripts/db_rule_engine.py --rule cancel_rate_spike` | Run single rule |
| `python3 scripts/db_export_feishu.py --all` | Export 5 Feishu CSVs |
| `python3 scripts/db_trigger_simulator.py --dry-run` | Read pending events, log only |
| `python3 scripts/db_trigger_simulator.py --apply` | Also update outbox status |
| `python3 scripts/db_status.py` | Show database status |
| `python3 scripts/run_db_pipeline.py --mode full` | Run all 6 steps sequentially |

## Troubleshooting

### Database not found
```bash
python3 scripts/db_init.py
```

### Data not ingested
```bash
python3 scripts/db_status.py
python3 scripts/db_ingest.py --mode full
```

### Duplicate run_ids
Pipeline uses timestamps for run_ids — re-runs create new entries. Safe to ignore.

### Rule engine not triggering
Check metric_daily has enough data points:
```bash
python3 -c "import sqlite3; c=sqlite3.connect('data/olist_ops.db'); print(c.execute('SELECT COUNT(*) FROM metric_daily').fetchone())"
```

### Tests failing
```bash
python3 -m pytest tests/test_db_*.py -v
```

## Database Schema

Database: `data/olist_ops.db` (NOT committed to Git)

12 tables across 4 categories:
1. **System**: pipeline_runs, ingestion_batches
2. **DWD**: dwd_order_level (99,441 rows), dwd_item_level (112,650 rows)
3. **Metrics**: metric_daily (~634 rows), metric_dimension_daily
4. **Decision**: alert_events, strategy_recommendations, action_tasks, review_retro
5. **Dispatch**: event_outbox, qoder_jobs
