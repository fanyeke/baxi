# v0.5 API Gateway Runbook

## Setup

```bash
pip install -r requirements.txt
echo 'API_BEARER_TOKEN=your-secret-token' >> .env
```

## Start

```bash
python3 scripts/run_api.py
# or with custom port:
python3 scripts/run_api.py --port 8765
```

API docs: http://127.0.0.1:8765/docs
OpenAPI: http://127.0.0.1:8765/openapi.json

## P0 Endpoints

### Health (no auth)
```bash
curl http://127.0.0.1:8765/api/v1/health
# {"status":"ok","version":"0.5.0","db_connected":true}
```

### Status (auth required)
```bash
curl -H "Authorization: Bearer dev-token" http://127.0.0.1:8765/api/v1/status
# Returns table counts + last pipeline run info
```

### Alerts (auth required)
```bash
curl -H "Authorization: Bearer dev-token" "http://127.0.0.1:8765/api/v1/alerts?severity=high&limit=10"
```

### Tasks (auth required)
```bash
curl -H "Authorization: Bearer dev-token" "http://127.0.0.1:8765/api/v1/tasks?status=todo&limit=10"
```

### Outbox (auth required)
```bash
curl -H "Authorization: Bearer dev-token" "http://127.0.0.1:8765/api/v1/outbox?status=pending&limit=10"
```

### Dispatch (auth required, dry-run by default)
```bash
# Dry-run (default)
curl -X POST -H "Authorization: Bearer dev-token" -H "Content-Type: application/json" \
  -d '{"channel":"manual","limit":1}' http://127.0.0.1:8765/api/v1/outbox/dispatch

# Apply (real execution)
curl -X POST -H "Authorization: Bearer dev-token" -H "Content-Type: application/json" \
  -d '{"channel":"manual","limit":1,"dry_run":false,"apply":true}' http://127.0.0.1:8765/api/v1/outbox/dispatch
```

## Auth

All non-/health endpoints require: `Authorization: Bearer <API_BEARER_TOKEN>`

Set in `.env` or env var: `API_BEARER_TOKEN=your-secret-token`

## Error Format

```json
{
  "request_id": "uuid",
  "error_code": "AUTH_REQUIRED",
  "message": "Authorization header is required",
  "diagnosis": "No Bearer token provided",
  "suggested_action": "Add 'Authorization: Bearer <token>' header"
}
```

## Troubleshooting

- **API_BEARER_TOKEN not set**: Add to `.env` or export env var
- **DB not found**: API warns on startup, health shows `db_connected: false`
- **Port in use**: Use `--port` flag on `run_api.py`
- **Schema migration**: Applied automatically at startup
