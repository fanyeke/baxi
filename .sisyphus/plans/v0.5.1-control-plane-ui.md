# v0.5.1: Control Plane UI — API P1 + 中台前端 Alpha

## TL;DR

> **Quick Summary**: 补齐 P1 API 端点（logs/feishu/pipeline）并构建 7 页 React 中台控制台，让 v0.5 从"API 后台"升级为"人类可观察、可诊断、可预演的控制台"。所有写操作默认 dry-run，前端仅通过 API 访问系统。

> - 3 个新 API 路由器：`api/routers/logs.py`, `feishu.py`, `pipeline.py`

> - OpenAPI 更新至 `docs/openapi-v0.5.1.json`
> - `pytest.ini` + `pytest-cov` 测试基础设施
> 
> **Estimated Effort**: Large
> **Parallel Execution**: YES — 8 waves, max 8 concurrent
> **Critical Path**: T1 → T2/T3/T4 → T5 → T6/T7/T8 → T9/T10/T11 → T12 → T22 → T24 → FINAL

## Context

### Original Request
v0.5 Local API Gateway 基线完成（6 个 P0 端点、Bearer 认证、229 测试）。下一步不做 Qoder 接入，改为控制台化：补齐 P1 API（logs/feishu/pipeline）+ 构建本地中台前端 Alpha。前端定位为系统控制台/数据中台界面，飞书是业务协同界面，本前端是系统控制台。

### Interview Summary
**Key Discussions**:
- P1 API: GET /logs/errors|audit|recent, POST /feishu/export|sync|status/import, POST /pipeline/run
- Logs 数据源：直接读 JSON 日志文件（tail-N 模式），不存 SQLite
- Feishu API：创建 `services/feishu_service.py` 封装 `feishu_client.py` 方法
- Pipeline/run：仅返回预览命令字符串，不实际执行子进程
- 前端：React + Vite + TS + TailwindCSS + shadcn/ui，7 页，纯 API 消费者
- 所有写操作默认 dry-run，apply 需二次确认
- 测试策略：TDD（RED-GREEN-REFACTOR）

**Research Findings**:
- 测试基础设施：pytest 7.0+, 145+ 测试, 最佳模式 `test_api_gateway.py`（TestClient + temp DB + monkeypatch）
- 日志：3 个 JSON logger（api/error/audit → `logs/api/`）+ 2 个 CSV 审计文件
- FeishuClient：9 个公共方法，全部 dry_run 感知，3 次重试backoff
- 现有路由器模式：`APIRouter(dependencies=[Depends(get_current_user)])` + Pydantic ListResponse
- 无 pytest.ini、无 CI、无覆盖率工具

### Metis Review
**Identified Gaps** (addressed):
- pytest.ini/pytest-cov 缺失：必须在写测试前创建 → T1
- services/log_reader.py 独立：不在 router 中直接读文件 → T2
- services/feishu_service.py：封装 FeishuClient → T3
- CORS：仅 dev（localhost:5173）→ T22
- 边缘情况：缺失日志文件 → 空列表；缺失 Feishu 凭据 → not_configured；部分 JSON 行损坏 → 跳过
- Pydantic v2 兼容：sqlite3.Row 需先转 dict → 现有模式已验证
- 前端认证：Bearer token 存 localStorage → 内部工具可接受

### Must Have
- 所有写端点默认 dry_run（apply 需显式传 true）

### Must NOT Have (Guardrails)
- **NO 新 Pydantic v1 模式** — 只用 Pydantic v2 `model_config = ConfigDict(from_attributes=True)`

- **NO 生产 CORS** — 仅 dev 模式允许 localhost:5173
- **NO 图表/仪表盘分析** — Dashboard 仅统计卡片，无图表
- **NO 实时同步/WebSocket/轮询** — 仅手动 POST 触发
- **NO 角色权限系统** — 所有认证用户同等访问
- **NO 前端直接连 SQLite 或调脚本**
