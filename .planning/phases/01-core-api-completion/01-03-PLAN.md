  <action>
    1. In internal/api/dto/outbox.go, add a BatchDispatchRequest struct with a single field:
       - DryRun bool `json:"dry_run"`
       This mirrors the BatchDispatchResponse.DryRun field for request symmetry.

    2. In internal/api/handler/outbox.go, replace the HandleBatchDispatch 501 stub (lines 192-194) with a full implementation:
       a. Parse optional request body: decode json.NewDecoder(r.Body).Decode(&req) where req is dto.BatchDispatchRequest. If body is empty or decode fails, default to dry_run=false. Do NOT return 400 on empty body — this is a fire-and-forget batch operation.
       
       b. Query pending events: construct model.OutboxFilters with Status="pending" (string pointer). Call h.svc.List(r.Context(), filters, 1000, 0) to get all pending events up to 1000.
       
       c. Initialize counters: dispatched=0, failed=0, eventIDs=[]string{}
       
       d. Iterate over resp.Items. For each event:
          - If req.DryRun is true, increment dispatched and append event.OutboxID to eventIDs, but do NOT call DispatchEvent.
          - If req.DryRun is false, call h.svc.DispatchEvent(r.Context(), event.OutboxID).
          - On DispatchEvent success: increment dispatched, append event.OutboxID.
          - On DispatchEvent error: increment failed, STILL append event.OutboxID (so caller knows it was attempted). Log the error but continue processing remaining events.
       
       e. Build and return BatchDispatchResponse:
          - DryRun: req.DryRun
          - Dispatched: dispatched count
          - Failed: failed count
          - EventIDs: collected event IDs
       
       f. Use httputil.JSON(w, http.StatusOK, resp)
       
       g. Error handling: if List fails, return 500 with "failed to list pending events". Individual dispatch failures are counted in Failed, not returned as HTTP errors.

    3. Ensure the existing BatchDispatchResponse struct (already defined at lines 184-189) is used as the response type.
  </action>
