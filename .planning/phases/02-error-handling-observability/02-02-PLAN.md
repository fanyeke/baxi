  <read_first>

  <action>
  修复两处静默忽略 JSON 解码错误的 bug（per ERR-03, D-03）：

  **action.go — HandleExecute（约第 56-66 行）：**
  将：
  ```go
  var req executeRequest
  if r.Body != nil {
      _ = json.NewDecoder(r.Body).Decode(&req)
  }
  ```
  改为：
  ```go
  var req executeRequest
  if r.Body != nil {
      if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
          writeError(w, r, http.StatusBadRequest, middleware.BAD_REQUEST, "invalid request body: "+err.Error())
          return
      }
  }
  ```
  关键点：保留 `if r.Body != nil` 检查——空 body 仍然允许（将使用默认的 dryRun=false），只有当解码真正失败时才返回 400。

  **outbox.go — HandleBatchDispatch（约第 193-198 行）：**
  将：
  ```go
  var req dto.BatchDispatchRequest
  if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
      // Empty body or invalid JSON defaults to dry_run=false
      req.DryRun = false
  }
  ```
  改为：
  ```go
  var req dto.BatchDispatchRequest
  if r.Body != nil {
      if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
          writeError(w, r, http.StatusBadRequest, middleware.BAD_REQUEST, "invalid request body: "+err.Error())
          return
      }
  }
  ```
  关键点：空 body → 使用零值（DryRun=false 是零值的默认值），invalid JSON → 返回 400。
  </action>

  - outbox.go HandleBatchDispatch: 当传入格式错误的 JSON 时返回 400，空 body 继续正常处理（默认 dry_run=false）
  - 现有 test 代码不受影响（`go build ./internal/api/...` 通过）
  - 使用 `_ = json.NewDecoder` 的模式在 action.go 和 outbox.go 中已彻底移除
