  **设计决策（执行者自行判断）：**
  - URL 解析失败时返回 false（fail closed），不跳过验证。记录一条 `log.Printf("...")` 但不需要引入 zap logger——`log` 标准库即可（现有代码风格：middleware 包目前无 logger 依赖）。
  - 使用 `net/url` 包解析 Origin 头和允许的来源字符串。

  **具体修改：**

  ### 1. 更新 import 块
  新增 import：`"net/url"`、`"net"`、`"log"`

  ### 2. 修改 `parseOrigins()` 签名
  返回类型从 `[]string` 改为 `[]url.URL`（值类型切片）。
  内部实现变更：
  - 使用 `url.Parse(p)` 解析每个 origin 字符串
  - 解析失败 → `log.Printf("invalid allowed origin %q: %v", p, err)` 后跳过该项（不添加到返回切片）
  - 解析成功后调用辅助函数 `normalizeHostPort(u)` 将 `u.Host` 重写为 `hostname:port` 格式，其中：
    - 使用 `u.Hostname()` 获取 host 部分
    - 使用 `u.Port()` 获取 port 部分
    - port 为空时：scheme 为 "https" 则使用 "443"，否则（含 "http"）使用 "80"
    - 使用 `net.JoinHostPort(hostname, port)` 组装
  - 将 `*u`（值）追加到结果切片

  ### 3. 修改 `isOriginAllowed()` 签名
  参数 `allowedOrigins []string` → `allowedOrigins []url.URL`
  内部实现变更：
  - 使用 `url.Parse(origin)` 解析传入的 Origin 头
  - 解析失败 → `log.Printf("failed to parse request origin %q: %v", origin, err)` → `return false`（fail closed）
  - scheme 或 host 为空 → `return false`
  - 调用 `normalizeHostPort(u)` 重写 `u.Host`
  - 遍历 `allowedOrigins`，比较：`u.Scheme == allowed.Scheme && u.Host == allowed.Host`
  - 任意匹配则 return true；全不匹配则 return false

  ### 4. 添加 `normalizeHostPort()` 辅助函数
  ```go
  // normalizeHostPort ensures the URL.Host includes an explicit port number.
  // If the port is missing, it assigns the default port for the scheme
  // (80 for http, 443 for https).
  func normalizeHostPort(u *url.URL) {
      host := u.Hostname()
      port := u.Port()
      if port == "" {
          switch u.Scheme {
          case "https":
              port = "443"
          default:
              port = "80"
          }
      }
      u.Host = net.JoinHostPort(host, port)
  }
  ```

  **不要修改：**
  - `NewCORSMiddleware()` 函数签名和整体结构（只改内部变量类型）
  - CORS 响应头格式（Access-Control-Allow-*）
  - 非 OPTIONS 请求的行为

  **不要做的：**
  - 不要引入新的依赖包（`net/url` 和 `net` 都是标准库）
  - 不要改变 `CORS_ALLOWED_ORIGINS` 的格式或解析方式
  - 不要在正常路径上添加 `log.Printf`（只对错误或异常情况记录日志）
